stockXchange


type :

Web Development

created in :

january 2023

Find it live here: stockxchange.onrender.com or Github Repository

A web app that allows you to manage portfolios of stocks and track their prices and portfolio values in real-time. You can access your portfolios on the go with its mobile compatibility and perform "buy" and "sell" transactions by querying an API for their prices. Built with Flask and hosted on Render, it uses Jinja templating engine, Bootstrap, and a bit of CSS for its client-side. The password is hashed for security, and the API is accessed via an environment variable. The app is a demo version using the AlphaVantage API and a local CSV file, but is limited by the API's usage limits. This project started as a Harvard CS50 course project and has taught the developer valuable lessons for future improvements, such as using more functions or Django for abstractions and improving data source through a better API or web scraping.

CLI pomodoro Timer


type :

Bash Scripting

created in :

january 2023

Find it here: github repo

CLI Pomodoro Timer is a simple command-line Pomodoro timer written in bash that allows you to quickly set a timer in the terminal. To use, make the script executable and run it with an integer argument representing the duration of the Pomodoro in minutes. The script will display a progress bar and once the timer is finished, display a system notification and play an alert sound. This script has been tested on Linux systems and uses the commands notify-send and paplay, which may not be present on other operating systems. Contributions to the project are welcome through pull requests or issues. The project is licensed under the MIT License.

Run World Cup Simulations


type :

Python Script

created in :

october 2022

Find it here: github repo

The code uses ELO rating system to determine the probability of a team winning a game and uses this information to simulate the entire tournament. The result of each simulation is stored in a dictionary and at the end, the chances of winning for each team are calculated based on the number of wins in the simulations. The project uses the main() function to run the simulation N number of times (default is 1000), where N is the number of times the tournament is simulated. The code has three additional functions: simulate_game(), simulate_round(), and simulate_tournament(). The final output is the chances of winning for each team, sorted by the highest chances of winning. This project can be run in a matter of milliseconds and the number of simulations can be increased to obtain more accurate results, but this would trade off with runtime.
